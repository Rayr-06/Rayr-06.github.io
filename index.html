05 · How I Think

Engineering philosophy

01
Domain expertise is the moat
Any developer can call a GPT API. What separates a good AI product from a generic one is deep understanding of the problem domain. Six years inside real software quality work tells you what an AI test tool actually needs to do — and what it will get wrong if you don't design for it.
02
Ship it. Then improve it.
Every project on this page is publicly accessible right now. Not a mockup, not a GitHub repo with a long README and no working code. Real deployment teaches what design thinking never does — what breaks under real users, real data, and real devices.
03
Show honest numbers
RAYR Fin shows real XIRR from actual purchase data — not an estimated percentage. QA Forge generates test cases from actual requirements — not templates. The product should tell the truth, even when the truth is harder to calculate than a guess.
04
Build for real people
Blur mode in RAYR Fin exists because I thought about someone using their phone in public. The FD comparison exists because I know what Indian investors always ask first. Empathy for the user is a technical requirement, not a soft skill.
"Quality is not a phase at the end of development. It is a system you design from the beginning — and the best systems prevent problems rather than finding them after the fact."
